# PRD · MLIGB1 — Biology Study Hub 迭代升级

> **分支**: `MLIGB1`
> **日期**: 2026-07-08
> **状态**: Draft → 待用户评审
> **基线**: unit2.html (5-Part 架构 v1.5)

---

## 1. 背景与问题

当前 Biology Study Hub（unit1/unit2）存在以下核心问题：

| # | 问题域 | 现状 | 影响 |
|---|--------|------|------|
| A | 词汇查词 | 内置 PHONETICS 字典（150+ 词），仅点击预置按钮发音，无划词查词 | 学生遇到生词无法即时查询释义和朗读 |
| B | 长句理解 | 课文原文为纯展示，无交互辅助 | 学生读不懂复杂长句，流失率高 |
| C | 主观题 | unit2 有 Short Answer 但只展示预设答案，无 AI 评价 | 无法检验学生真实理解程度 |
| D | 康奈尔笔记 | 纯英文，字体层级弱 | 中文母语学生难以快速抓住要点 |
| E | 导航 | 只有顶部 Tab，无侧边目录/进度条 | 长页面找不到位置 |
| F | 知识框架 | 无 | 学生被动阅读，缺少主动建构 |
| G | 错题本 | 存在但名称隐蔽（"错题本"），功能不突出 | 学生不知道有这功能 |
| H | 架构 | 单体 HTML，TTS 依赖本地 Python 代理 | 稳定性差，部署复杂 |
| I | 扩展性 | 内容硬编码在 HTML 中 | 无法动态补充资料 |

## 3. 需求清单（11 项）

### R1 · 智能划词查词 + 朗读
**优先级**: P0

- 用户在页面任意位置选中文本（单词/短语/句子），弹出浮窗显示：
  - 英文释义 + 中文翻译
  - IPA 音标
  - 🔊 朗读按钮（整句朗读）
- **后端方案**: 接入 DeepSeek API 作为查词/翻译引擎（前端划选 → 后端查 → 返回释义）
- **降级方案**: 如果 DeepSeek 不可用，回退到内置字典 + Web Speech API
- 参考: unit1 的 `PHONETICS` 字典和 `speakWord()` 函数可复用

### R2 · 长句划线解释 + 一键收藏（v2 升级 2026-07-09）
**优先级**: P0

- 用户选中任意文本（单词/短语/句子/段落），浮窗提供：
  - 🔊 **朗读** — 整句/单词朗读
  - 💡 **解释** — 调用 GLM API 返回：句子主干（主谓宾）、关键术语标注、中文翻译、逻辑解释
  - ➕ **大加号按钮** — 将选中内容 + AI 解释一键加入笔记本
    - 收藏内容包含：原文、释义/解释、来源位置（哪个 Part/Section）
    - 如果页面有相关图片，一并收藏（截图 canvas）
    - 加号按钮视觉醒目（大号圆形，绿色/紫色渐变，带动画反馈）
- 结果以折叠卡片形式展示在浮窗内，"已加入笔记本"后有 toast 动画确认

### R3 · 主观问答 + AI 评价
**优先级**: P0

- 将现有 Short Answer 从"展示答案"升级为"作答 → AI 评价"
- 流程：
  1. 学生在文本框中手写答案
  2. 提交后调用 DeepSeek API 评分（对标 IGCSE 评分标准）
  3. 返回：分数 + 详细解释（哪里对、哪里错、怎么改进）
  4. 提供"跳转课文"按钮，定位到相关段落
- 每次作答记录到笔记本

### R4 · 康奈尔笔记中英混合优化（v2 升级 2026-07-09）
**优先级**: P1

- **加宽布局**: Cornell 主区域宽度从当前扩展到 100%（去掉右侧 cue 列或改为折叠式），正文区占满
- **字体层级强化**（更大区分度）：
  - Section 大标题: **22px bold**，带底部双线
  - 中文主题概要: **14px medium**，黄色高亮背景，每段开头
  - 英文段落: **16px regular**，行高 2.0（舒适阅读）
  - 关键术语行: **15px semibold + 下划线**
  - 生物名词中文标注: 内联显示，用括号和颜色区分
- **增加更多中文标注**：
  - 每个段落开头加 1-2 句中文导读
  - 关键术语后内联标注中文（如 cytoplasm（细胞质））
  - 重要概念用中文 callout 补充解释
- 示例：
  ```
  ┌──────────────────────────────────────────────┐
  │ 2.1 Organisms are made up of cells           │  22px bold + 双线
  │ 📌 细胞的基本结构 — 所有生物都由细胞构成       │  14px 黄底
  │                                              │
  │ 细胞各部分的结构和功能如下：                   │  14px 中文导读
  │                                              │
  │ The **cytoplasm**（细胞质）is surrounded by  │  16px + 内联中文
  │ the **cell surface membrane**（细胞膜）.     │
  └──────────────────────────────────────────────┘
  ```

### R8 · 前后端分离架构 + RAG（v3 最终选型 2026-07-09）
**优先级**: P2（架构层，支撑 R1-R3, R9）

- **部署方案**: Vercel（前端 + API Routes）+ Supabase（数据库 + 向量存储 + Auth）
  - 费用：**$0**（两者免费档均够用）
  - 部署流程：`git push` → Vercel 自动构建部署
- **前端**: 纯静态 HTML/JS（现有），部署到 Vercel
- **后端**: Vercel API Routes（Node.js Serverless Functions）
  - `/api/lookup` — 划词查词/翻译（DeepSeek）
  - `/api/explain` — 长句解释（DeepSeek）
  - `/api/grade` — 主观题评分（DeepSeek）
  - `/api/rag/query` — RAG 问答（Supabase 向量检索 + DeepSeek 生成）
  - `/api/rag/upload` — 上传补充资料（浏览器端 pdf.js 解析 + 智谱 embedding）
  - `/api/vision` — 图片理解（GLM-4V）
- **大模型分工**:
  | 任务类型 | 模型 | 原因 |
  |---------|------|------|
  | 文本理解/生成 | **DeepSeek-V3** | ¥1/百万 token，便宜 |
  | 深度推理评分 | **DeepSeek-R1** | ¥4/百万 token |
  | 视觉/看图 | **GLM-4.6V-Flash** | 免费（Coding Plan） |
  | Embedding | **智谱 embedding-3** | DeepSeek 无嵌入模型 |
- **API Key 管理**: 全部存 Vercel 环境变量，前端不接触 Key
- **RAG 架构**（v3 — Supabase pgvector）:
  ```
  课文/资料 → 浏览器 pdf.js 提取文本 → 智谱 embedding-3 → Supabase pgvector
                                                                      ↓
  学生提问 → 智谱 embedding-3 → Supabase 余弦相似度 Top-K=5 → DeepSeek 生成回答
  ```
  - **向量数据库**: Supabase pgvector（云端、免费、支持 SQL 混合检索）
  - **Embedding 模型**: 智谱 embedding-3
  - **回答生成**: DeepSeek-V3 + 课文上下文，附来源引用
  - 浏览器内嵌聊天窗口，学生可随时提问
- **为什么不用 FastAPI + ChromaDB + Railway**:
  - Supabase 一个服务替代 ChromaDB + 数据库 + Auth + Storage
  - Vercel 免费 + 全球 CDN，Railway 美国节点国内慢
  - Serverless 冷启动 ~1s 对 AI 场景影响可忽略

### R15 · Cornell 笔记英文化重构（2026-07-09 新增）
**优先级**: P1（插排在任务1和任务2之间）

**问题**: 当前 Cornell 侧栏中英文混合，中文占比过高；正文也出现大量中文，不利于英语沉浸式学习。

**目标**: 营造全英文阅读环境，侧栏只保留少量关键中文提示，正文纯英文。

- **侧栏（cue column）调整**:
  - 现有中文摘要质量好 → 保留内容，但**翻译成英文**
  - 仅保留少量关键中文术语提示（如 `cell wall (细胞壁)` 格式）
  - 英文做对应段落/意群的核心意思概要（summary）
- **正文（main column）调整**:
  - 正文部分**尽量不出现中文**
  - 现有的 `cn-intro`（中文导读）改为英文导读
  - 现有的 `cn-callout`（记忆提示）改为英文，保留视觉效果
  - 现有的 `cn-brief` 改为英文摘要
  - 现有的 `term-inline` 内联中文标注 → 去掉中文，保留英文加粗
- **保留元素**:
  - 图片和图注保持不变（图片讲解已经是独立的 R14b 功能）
  - CSS 类名保持不变（仅改内容），减少代码变动

### R16 · Cornell 笔记动态扩展机制（2026-07-09 新增）
**优先级**: P2（PM 研究项）

**问题**: Cornell 笔记当前是硬编码的静态内容，无法动态扩展。

**目标**: 建立 Cornell 笔记的动态补充机制，支持多来源内容注入。

- **扩展来源**:
  1. **错题提示**: 学生做错题后，相关知识点自动在 Cornell 对应段落添加提示标记
  2. **补充资料**: R9 上传的资料中与课文相关的内容，自动建议补充到 Cornell
  3. **作业重点**: 老师提交的作业考察重点，标注在 Cornell 相关段落
- **实现方向（待 PM 确认）**:
  - 数据结构: Cornell 内容从硬编码迁移到结构化数据（JSON）
  - 动态注入: 每个 Cornell 段落支持 `annotations[]` 数组，存放来自不同来源的补充
  - 视觉标记: 不同来源用不同颜色标记（蓝=补充资料，红=错题提示，金=作业重点）
  - 审核流程: AI 建议的补充需要学生确认后才显示

### R9 v2 · 补充资料上传 + GLM 视觉 MCP 全流程（2026-07-09 升级）
**优先级**: P2

**升级要点**: 上传资料 → GLM 视觉 MCP 识别 → 自动扩充三个维度。

- **支持格式**: PDF, PNG, GIF, Apple HEIC/HEIF（苹果照片原格式保留，不做转换）
- **处理流程**:
  ```
  用户上传资料（PDF/PNG/GIF/HEIC）
    ↓
  ① GLM 视觉 MCP 识别内容（每份资料都跑一遍 AI）
    ├── 图片: GLM-4.6V 直接理解
    ├── PDF: 逐页截图 → GLM-4.6V 识别
    └── HEIC: 转为 base64 → GLM-4.6V 识别
    ↓
  ② AI 输出结构化结果（三层扩充）
    ├── 词条层: 识别新术语 → 建议加入 Part 1 词条库
    ├── 阅读层: 提取关键段落 → 建议补充到课文阅读区
    └── Cornell层: 提取与课文相关的知识点 → 建议补充到 Cornell 笔记
    ↓
  ③ 用户选择录入内容（审核界面）
    ├── 勾选要采用的词条/段落/Cornell 补充
    ├── 编辑修改 AI 建议的内容
    └── 确认后入库（写入 Supabase + 更新前端）
  ```
- **录入选择界面**:
  - 上传后弹出审核面板，分为三个 Tab：词条 / 阅读 / Cornell
  - 每个 Tab 显示 AI 识别的内容，可勾选/编辑/拒绝
  - 确认后自动添加到对应 Part
- **原格式保留**: HEIC/HEIF 照片不做格式转换，原样存储到 Supabase Storage
