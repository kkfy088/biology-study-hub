## 1. 背景与问题

当前 Biology Study Hub（unit1/unit2）存在以下核心问题：

| # | 问题域 | 现状 | 影响 |
|---|--------|------|------|
| A | 词汇查词 | 内置 PHONETICS 字典（150+ 词），仅点击预置按钮发音，无划词查词 | 学生遇到生词无法即时查询释义和朗读 |
| B | 长句理解 | 课文原文为纯展示，无交互辅助 | 学生读不懂复杂长句，流失率高 |
| C | 主观题 | unit2 有 Short Answer 但只展示预设答案，无 AI 评价 | 无法检验学生真实理解程度 |
| D | 康奈尔笔记 | 纯英文，字体层级弱 | 中文母语学生难以快速抓住要点 |
| E | 导航 | 只有顶部 Tab，无侧边目录/进度条 | 长页面找不到位置 |
| F | 知识框架 | 无 | 学生被动阅读，缺少主动建构 |
| G | 错题本 | 存在但名称隐蔽（"错题本"），功能不突出 | 学生不知道有这功能 |
| H | 架构 | 单体 HTML，TTS 依赖本地 Python 代理 | 稳定性差，部署复杂 |
| I | 扩展性 | 内容硬编码在 HTML 中 | 无法动态补充资料 |

- **Michelle**（初二，中文母语）— 使用 IGCSE 0610 Biology 课程学习
- 场景：课后自学、考前复习、遇到生词和长句需要即时辅助

## 3. 需求清单（11 项）

### R1 · 智能划词查词 + 朗读
**优先级**: P0

- 用户在页面任意位置选中文本（单词/短语/句子），弹出浮窗显示：
  - 英文释义 + 中文翻译
  - IPA 音标
  - 🔊 朗读按钮（整句朗读）
- **后端方案**: 接入 DeepSeek API 作为查词/翻译引擎（前端划选 → 后端查 → 返回释义）
- **降级方案**: 如果 DeepSeek 不可用，回退到内置字典 + Web Speech API
- 参考: unit1 的 `PHONETICS` 字典和 `speakWord()` 函数可复用

### R2 · 长句划线解释
**优先级**: P0

- 用户选中一个长句，浮窗增加"解释"按钮
- 点击后调用 DeepSeek API 分析句子结构，返回：
  - 句子主干（主谓宾）
  - 关键术语标注
  - 中文翻译
  - 为什么这么说的逻辑解释（1-2 句）
- 结果以折叠卡片形式展示在浮窗内

### R3 · 主观问答 + AI 评价
**优先级**: P0

- 将现有 Short Answer 从"展示答案"升级为"作答 → AI 评价"
- 流程：
  1. 学生在文本框中手写答案
  2. 提交后调用 DeepSeek API 评分（对标 IGCSE 评分标准）
  3. 返回：分数 + 详细解释（哪里对、哪里错、怎么改进）
  4. 提供"跳转课文"按钮，定位到相关段落
- 每次作答记录到笔记本

### R4 · 康奈尔笔记中英混合优化
**优先级**: P1

- 每段课文笔记增加**中文要点标注**（1-2 句话概括这段在讲什么）
- 字体层级强化：
  - 大标题: 20px bold
  - 小节标题: 16px semibold
  - 英文原文: 14px regular
  - 中文标注: 13px，颜色用 `var(--accent)` 区分
- 示例：
  ```
  [2.1 Cell Structure] (20px bold)
    细胞结构 — 讲细胞各部分的名字和功能 (13px accent)

  The cytoplasm is surrounded by... (14px regular)
  ```

### R5 · 页面导航增强
**优先级**: P1

- 左侧或顶部增加**粘性侧边栏**，包含：
  - 5-Part 快速跳转（① ② ③ ④ ⑤）
  - 当前位置高亮
  - 学习进度条（已完成几个 Part）
- 滚动时自动高亮当前 Part

### R6 · 知识框架填空
**优先级**: P1

- 新增 Part 6: "知识框架"（Knowledge Frame）
- 展示本单元的核心知识结构图（树状/表格），关键节点留空
- 学生填空提交后：
  - 自动判分（精确匹配 + 近义匹配）
  - 错误处标红，给出提示
  - 记录到笔记本
- 示例：
  ```
  细胞 → [_____] → 控制物质进出
         [_____] → 产生能量（呼吸作用）
         [_____] → 合成蛋白质
  ```

### R7 · 笔记本（原"错题本"升级）
**优先级**: P1

- 名称从"错题本"改为**"笔记本"** 📒
- 位置更醒目：右侧浮动按钮 + Badge 数字
- 功能整合：
  - 🚩 生词本（来自 Part 1）
  - ❌ 错题（来自 Part 2/5/6）
  - 📝 主观题作答记录（来自 Part 5）
  - ⭐ 收藏的句子/段落
- 支持 PDF 导出（已有功能保留）

### R8 · 前后端分离架构 + RAG
**优先级**: P2（架构层，支撑 R1-R3, R9）

- **前端**: 纯静态 HTML/JS，部署到 GitHub Pages
- **后端**: 轻量 API 服务（Python FastAPI / Node.js），提供：
  - `/api/lookup` — 划词查词/翻译（R1）
  - `/api/explain` — 长句解释（R2）
  - `/api/grade` — 主观题评分（R3）
  - `/api/rag/query` — RAG 问答（R8b）
  - `/api/rag/upload` — 上传补充资料（R9）
- **AI 引擎**: 智谱 GLM-4.6V（Coding Plan，4000 次/月）
  - 文本理解：GLM-4.6 或 GLM-4-Flash（免费）
  - 视觉理解：GLM-4.6V-Flash（免费）或 Coding Plan 额度
  - 后期可切换至 GPT-4o（vision-mcp 支持热切换，改 3 行配置即可）
- **RAG 组件**:
  - 课文 PDF + 补充资料 → 向量数据库（ChromaDB / FAISS）
  - 浏览器内嵌聊天窗口，学生可随时提问
  - 回答附带来源引用（哪页课文/哪段）
- **部署方案**: 后端部署到 Vercel/Railway/本地，前端通过 fetch 调用

### R9 · 补充资料上传 + 视觉解析
**优先级**: P2

- 学生/老师可上传补充资料（PDF/Word/图片）
- **视觉解析方案**（2026-07-08 确定）：
  - OCR 层：macos-vision-mcp（本地，免费，省 97% token）
  - 视觉理解层：GLM-4.6V via vision-mcp（@haisto/vision-mcp）
    - 生物插图理解：GLM-4.6V 原生中文优势
    - 几何辅助线分析：GLM-4.1V-Thinking（思考链模式）
    - 物理示意图理解：GLM-4.6V VQA 能力
  - 后期升级路径：切换 OPENAI_MODEL 为 gpt-4o 即可
- 上传后：
  1. 文本提取（macos-vision OCR）→ 入向量数据库
  2. 图片理解（GLM-4.6V）→ 自动识别新术语 → 建议加入词条库
  3. 自动识别与课文相关的段落 → 建议补充到对应 Part
- 需要审核流程：AI 建议 → 用户确认 → 入库
